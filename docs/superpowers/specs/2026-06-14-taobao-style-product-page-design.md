# 淘宝风格产品页面改造设计

## 概述

将现有的弹窗式产品新增/编辑改为全页分区块表单，将现有的弹窗式产品详情查看改为全屏淘宝宝贝风格页面。采用组件级视图切换方案（不引入路由库）。

## 需求确认

| 需求 | 决策 |
|------|------|
| 主图和详情图层级 | 产品级别，所有 SKU 共享 |
| 主图数量上限 | 最多 10 张 |
| 详情图数量上限 | 不限 |
| SKU 级别图片 | 保留（SKU图、尺寸图） |
| 新增产品页面布局 | 全页表单，分区块纵向排列 |
| 产品详情页 | 全屏页面跳转（非弹窗） |
| 架构方案 | 组件级视图切换（无路由库） |

## 数据库变更

### products 表新增字段

```sql
ALTER TABLE products ADD COLUMN main_images TEXT;   -- JSON array of image URLs, max 10
ALTER TABLE products ADD COLUMN detail_images TEXT; -- JSON array of image URLs, unlimited
```

### 迁移逻辑

- 检测 `main_images` 和 `detail_images` 字段是否存在，不存在则添加
- 默认值为空 JSON 数组 `[]`

## 后端 API 变更

### 产品接口 payload 扩展

`POST /api/products` 和 `PUT /api/products/:id` 的 body 新增：

```json
{
  "main_images": ["/uploads/img1.jpg", "/uploads/img2.jpg"],
  "detail_images": ["/uploads/detail1.jpg", "/uploads/detail2.jpg"]
}
```

### 产品查询返回

`GET /api/products` 和 `GET /api/products/:id` 返回的产品对象新增 `main_images` 和 `detail_images` 字段（解析后的数组）。

## 前端架构

### App.tsx 视图状态

```tsx
type Page = 
  | { type: 'dashboard' }
  | { type: 'product-new' }
  | { type: 'product-edit', productId: number }
  | { type: 'product-detail', productId: number };
```

App 组件根据 `currentPage` 状态渲染对应视图，不使用路由库。

## 边界情况

1. **主图为空**：详情页主图区域显示占位图
2. **详情图为空**：不显示"商品详情"区块
3. **只有一个 SKU**：规格选择器隐藏，直接显示该 SKU 信息
4. **编辑旧数据**：main_images/detail_images 为空时显示为空数组
5. **图片上传失败**：显示错误提示，不添加到列表
